# Formal Style Guide

Professional and structured. Elevates register, removes colloquialisms.

## Characteristics

- **Register**: High, professional
- **Tone**: Objective, authoritative
- **Structure**: Clear, logical organization
- **Language**: Precise, technical where appropriate

## When to Use

- Academic papers
- Business reports
- Official documentation
- Professional communications

## Polish Approach

### Do
- Use complete sentences
- Avoid contractions
- Replace colloquialisms with formal equivalents
- Use precise terminology
- Maintain objective tone
- Structure arguments logically

### Don't
- Use slang or casual expressions
- Use first person unless necessary
- Include emotional language
- Be verbose for formality's sake

## Example Transformations

### Colloquial to Formal
**Before**: "You can't just wing it when designing APIs."

**After**: "Ad-hoc API design approaches are insufficient for production systems."

### Contractions
**Before**: "It's important to test before deploying."

**After**: "It is essential to complete testing prior to deployment."

### Casual to Professional
**Before**: "This bug is a real pain—it messes up the whole workflow."

**After**: "This defect significantly disrupts the established workflow."

### First to Third Person
**Before**: "I think we should consider alternative approaches."

**After**: "Alternative approaches warrant consideration."

## Formal Language Substitutions

| Casual | Formal |
|--------|--------|
| can't | cannot |
| don't | do not |
| it's | it is |
| there's | there is |
| a lot of | numerous, substantial |
| big | significant, considerable |
| good | effective, satisfactory |
| bad | inadequate, problematic |
| thing | element, factor, component |
| get | obtain, acquire, receive |
| show | demonstrate, indicate, illustrate |
| use | utilize, employ, implement |
| need to | must, requires |
| want to | intends to, seeks to |

## Prompt Additions

When using formal style, add to polish prompt:

```
Style: Formal - Professional and structured
- Use complete sentences without contractions
- Replace colloquialisms with formal equivalents
- Use precise, technical terminology
- Maintain objective, authoritative tone
- Prefer third person over first person
- Structure arguments logically and clearly
- Avoid emotional or casual language
```

# Natural Style Guide

Smooth, natural flow that fixes awkward phrasing while preserving the author's unique voice.

## Characteristics

- **Flow**: Smooth, effortless reading
- **Voice**: Preserves author's personality
- **Rhythm**: Natural sentence variation
- **Clarity**: Clear without being simplistic

## When to Use

- General blog posts
- Most technical articles
- When author has a distinct voice to preserve
- Default choice when unsure

## Polish Approach

### Do
- Fix awkward phrasing and word order
- Smooth out jarring transitions
- Clarify ambiguous statements
- Vary sentence length for rhythm
- Maintain author's vocabulary level

### Don't
- Change the author's voice or personality
- Over-simplify complex ideas
- Remove author's unique expressions
- Make it sound generic

## Example Transformations

### Awkward Flow
**Before**: "The system architecture which was designed by the team last year and has been running in production since then, it handles about 1000 requests per second."

**After**: "The team designed this system architecture last year. It has been running in production ever since, handling about 1,000 requests per second."

### Jarring Transition
**Before**: "Python is easy to learn. Java is used in enterprise."

**After**: "While Python is known for being easy to learn, Java remains the go-to choice for enterprise applications."

### Ambiguous Statement
**Before**: "This approach is better for performance."

**After**: "This approach reduces latency by 40% compared to the previous implementation."

## Prompt Additions

When using natural style, add to polish prompt:

```
Style: Natural - Smooth, effortless flow while preserving author voice
- Fix awkward phrasing and word order issues
- Smooth transitions between ideas
- Maintain author's vocabulary and tone
- Vary sentence structure for natural rhythm
- Preserve unique author expressions and personality
```
